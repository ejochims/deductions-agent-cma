How it works

The tool boundary is the security boundary

The agent runs on Anthropic's orchestration layer. Its six tools are declared as custom tools but fulfilled host-side by the orchestrator, not inside the container. When the agent calls a tool, the session emits custom_tool_use and idles; the host runs the lookup against fixtures/ and returns a custom_tool_result. Nothing is mounted into the sandbox — no egress, no filesystem.

Why the numbers are trustworthy

The harness is validated before any result is believed

Five graders

Every draft is scored by deterministic checks: action correct, amount within tolerance, evidence cited, no hallucinated evidence, and a safety hard-fail — threshold respected (never settle > $10k).

Two calibration gates

Gate A: all 18 hand-written reference solutions score perfect. Gate B: a blind "approve-everything" agent fails every judgement bucket. Known-good perfect + known-bad at the floor → the ruler measures what it claims.

A calibrated judge

The LLM judge must fail three planted bad justifications before it is allowed to grade anything. It records rationale quality but never overrides the programmatic pass/fail.

pass^k reads harder than an average: a case counts as passing only if all three trials got it right — the metric that matters when the failure mode is a confident wrong settlement.

Stress-testing the prompt

Which prompt sections are load-bearing?

Each experiment weakens one section of the system prompt, re-runs the eval, then reverts. If the score drops, that section was doing real work — and the grader that catches the drop is named. If it doesn't drop, that's an honest negative worth keeping.

Load-bearing

What generalizes

Three design decisions worth stealing

The tool boundary is the security boundary

Because nothing is mounted, the answer key can't leak into context and the untrusted remittance text can't reach the filesystem. Access control is a property of the architecture, not a prompt instruction.

Policy lives in code, not the prompt

The $10k threshold and the null-amount coercion on a deny are enforced in the tool layer. The model can be wrong or adversarially steered and the money still can't move — the worst case is a bad draft a human reviews.

Validation is a first-class artifact

Graders, a calibrated judge, calibration gates, and a regression harness ship alongside the agent. That is the difference between a plausible demo and a defensible one — and what lets a finance owner sign off on auto-settling under a threshold.
